## 1.1 Introduction

Legal contracts are the foundation of modern commerce, employment, and civil agreements. Yet, for millions of individuals—students, freelancers, small business owners, and first-time renters—signing a contract is often an act of blind trust. The language is dense, the clauses are technical, and the implications are rarely obvious. A single overlooked clause can mean the difference between a fair agreement and a deeply unfavourable one.

The ContractShield: Smart Contract Review and Risk Analyzer project was born from this real-world problem. As a final-year project for the Bachelor of Science in Computer Science, its primary motivation is to build a practical, AI-powered legal technology tool that democratizes access to contract understanding. The core idea is to combine Natural Language Processing (NLP), rule-based legal analysis, and generative AI to build a unified platform that any user can interact with—regardless of their legal background.

ContractShield is a comprehensive web application that enables users to upload legal contracts (in PDF format), have them automatically parsed and analyzed, and receive a detailed breakdown of the document's risks, clauses, and compliance status. The platform performs clause extraction using the spaCy NLP library, evaluates risk using a weighted keyword scoring engine across fifty-plus legal risk indicators, checks compliance against the Indian Contract Act, 1872, and generates plain-English summaries using the Groq LLM API (llama-3.3-70b model).

Beyond document analysis, ContractShield includes an AI Legal Agent for conversational legal assistance, a lawyer marketplace where users can discover and book verified legal professionals, in-app real-time chat between clients and their appointed lawyers, WebRTC-powered video/audio call functionality for remote consultations, a community discussion forum, and a robust admin panel for platform governance.

The system is deployed in production on Railway (backend) and Vercel (frontend), making it a fully live, accessible, and scalable application.

---

## 1.2 Objective

The principal objective of the ContractShield project is to design and develop a comprehensive, AI-powered, and user-friendly web application that simplifies and enhances the process of legal contract review for non-experts. This overarching goal is broken down into the following specific, functional objectives:

- **To Automate Contract Analysis Using AI/NLP:** The platform must parse uploaded PDF contracts, extract text, identify and categorize legal clauses using spaCy NLP, and produce a structured, machine-readable analysis of the document's contents.

- **To Provide Quantified Risk Assessment:** A core objective is to compute a weighted risk score for each contract based on fifty-plus pre-defined legal risk keywords and clause patterns. The score must be accompanied by a risk level (Low / Medium / High / Critical) and a per-clause breakdown.

- **To Check Legal Compliance:** The system must evaluate contracts against the Indian Contract Act, 1872, flagging missing essential clauses (e.g., consideration, lawful object, mutual consent) and highlighting potentially non-compliant provisions.

- **To Generate Plain-English Summaries via LLM:** To bridge the gap between legal professionals and lay users, the platform must integrate the Groq LLM API to generate human-readable, jargon-free summaries of complex contracts.

- **To Provide an AI Legal Agent for Conversational Guidance:** A dedicated AI Legal Agent must allow users to ask free-form legal questions and receive contextual, informed responses in a chat interface.

- **To Build a Verified Lawyer Marketplace:** The platform must include a curated directory of verified lawyers with profiles, specializations, experience, bar council IDs, and hourly rates—allowing clients to discover and book legal professionals.

- **To Enable Real-Time Communication:** Approved appointments must unlock real-time in-app chat (via REST polling) and WebRTC-based audio/video calls between clients and their lawyers, enabling full remote legal consultation.

- **To Ensure a Secure and Personalized User Experience:** The system must feature JWT-based user authentication, role-based access control (client, lawyer, admin), and an admin approval workflow for new lawyer registrations.

- **To Foster Community Learning via Discussion Forum:** A community discussion board must allow users to post questions, vote on posts, and comment—creating a peer-to-peer legal knowledge base.

## 1.4 System in Use

ContractShield is a modern, full-stack web application. It does not require any specialized hardware from the end-user beyond a standard computer with internet access.

**Hardware Requirements:**
- A desktop or laptop computer with a modern processor and at least 4 GB of RAM.
- A stable internet connection for accessing the deployed web application.
- A standard monitor, keyboard, and mouse (webcam and microphone required for video call features).

**Software Requirements:**

- **Operating System:** The platform is OS-agnostic and is accessible from Windows, macOS, or Linux via any modern web browser.

- **Web Browser:** A current version of Google Chrome, Mozilla Firefox, Microsoft Edge, or Safari is required.

- **Backend Technology:** The server-side is built using **Python 3.11** and the **Flask** micro-framework. The ORM is **SQLAlchemy** (Flask-SQLAlchemy), database migrations are managed by **Flask-Migrate** (Alembic), authentication uses **Flask-JWT-Extended**, and real-time communication is powered by **Flask-SocketIO**. PDF text extraction utilizes **PyMuPDF (fitz)**. NLP clause extraction is performed by **spaCy** (en_core_web_sm model). Risk summarization and the AI Legal Agent are powered by the **Groq API** (llama-3.3-70b-versatile model). The production server is **Gunicorn** (gthread worker class).

- **Frontend Technology:** The user interface is built with **React 18** (Vite build tool), using **Vanilla CSS** for styling, **Axios** for HTTP requests, **Socket.IO-client** for real-time events, **Lucide-React** for icons, and the browser's native **WebRTC API** for video/audio calls.

- **Database:** The production database is **PostgreSQL**, hosted on Railway's managed database service. The application connects via a public TCP proxy URL, with SQLAlchemy's connection pool configured with TCP keepalives for reliability.

- **Deployment Environment:** The backend is deployed on **Railway** (containerized, auto-deployed from GitHub). The frontend is deployed on **Vercel** (static build, auto-deployed from GitHub). Environment variables (API keys, database URLs, JWT secrets) are managed through each platform's secure secrets management.

## 4.2 Module Description

ContractShield is built as a modular Flask application. Each major feature is encapsulated within a dedicated Blueprint, ensuring the system is organized, maintainable, and independently scalable.

---

### 1. User Authentication & Authorization Module (`app/blueprints/auth`)

**Purpose:** Handles all aspects of user identity, secure session management, and role-based access control using JSON Web Tokens (JWT).

**Key Features:**
- **Registration:** Users register as client or lawyer. Lawyers submit additional profile fields (specialization, experience, bar council ID). Lawyer accounts enter a pending state awaiting admin approval.
- **Login / Logout:** Password verification using `werkzeug.security`. On success, a JWT access token is issued and stored in the client's localStorage.
- **JWT Authentication:** All protected routes require a valid Bearer token via `@jwt_required()`. The `get_jwt_identity()` function retrieves the current user's ID.
- **Admin Seeding:** The first user with the designated admin email is automatically granted admin privileges at application startup.
- **Role-Based Access Control:** Decorators and route guards ensure clients, lawyers, and admins only access their respective resources.

---

### 2. Contract Upload & Management Module (`app/blueprints/contracts`)

**Purpose:** Manages the complete lifecycle of a legal contract—from secure upload to full AI/NLP analysis.

**Key Features:**
- **Secure PDF Upload:** Accepts PDF files up to a configurable size limit. Files are stored in a server-side uploads directory with UUID-generated filenames to prevent collisions.
- **Text Extraction:** Uses **PyMuPDF (fitz)** to extract full text from uploaded PDFs, including metadata like page numbers for clause location tracking.
- **NLP Clause Extraction (spaCy):** The `NLPService` uses the `en_core_web_sm` spaCy model to perform named entity recognition, sentence segmentation, and pattern-based clause classification across categories: Liability, Indemnification, Confidentiality, Termination, Payment, Jurisdiction, Force Majeure, IP Rights, and more.
- **Risk Scoring Engine:** The `RiskService` evaluates the contract using a weighted keyword dictionary of fifty-plus legal risk indicators. Each keyword match contributes to an aggregate weighted risk score, producing a final score (0–100) and risk level (LOW / MEDIUM / HIGH / CRITICAL).
- **Compliance Checking:** The `ComplianceService` evaluates the contract against the Indian Contract Act, 1872—checking for the presence of free consent, lawful consideration, lawful object, competent parties, and clarity of terms.
- **Missing Clause Detection:** Identifies legally important clauses that are absent from the contract (e.g., no governing law clause, no dispute resolution mechanism).
- **LLM Summary Generation:** The `GroqService` sends a structured contract prompt to the Groq API (llama-3.3-70b-versatile model) and returns a concise, plain-English summary of the contract's key terms and risks.

### `backend/app/services/nlp_service.py` — NLP Clause Extraction

```python
import spacy
import re

CLAUSE_PATTERNS = {
    'liability':        [r'liable', r'liability', r'damages', r'indemnif'],
    'termination':      [r'terminat', r'cancell', r'expir'],
    'confidentiality':  [r'confidential', r'non-disclosure', r'NDA', r'proprietary'],
    'payment':          [r'payment', r'invoice', r'fee', r'compensation'],
    'jurisdiction':     [r'jurisdiction', r'governing law', r'arbitration'],
    'force_majeure':    [r'force majeure', r'act of god', r'unforeseen'],
    'ip_rights':        [r'intellectual property', r'copyright', r'patent'],
}

class NLPService:
    def __init__(self):
        try:
            self.nlp = spacy.load('en_core_web_sm')
        except OSError:
            print('[NLPService] Warning: spaCy model not found. NLP disabled.')
            self.nlp = None

    def extract_clauses(self, text):
        if not self.nlp:
            return []
        doc = self.nlp(text)
        clauses = []
        for sent in doc.sents:
            sent_text = sent.text.strip()
            if len(sent_text) < 20:
                continue
            for clause_type, patterns in CLAUSE_PATTERNS.items():
                for pattern in patterns:
                    if re.search(pattern, sent_text, re.IGNORECASE):
                        clauses.append({
                            'clause_type': clause_type,
                            'content':     sent_text,
                            'risk_level':  self._assess_clause_risk(clause_type)
                        })
                        break
        return clauses

    def _assess_clause_risk(self, clause_type):
        HIGH_RISK = {'liability', 'termination', 'ip_rights'}
        MED_RISK  = {'confidentiality', 'jurisdiction'}
        if clause_type in HIGH_RISK: return 'HIGH'
        if clause_type in MED_RISK:  return 'MEDIUM'
        return 'LOW'
```

### `backend/app/services/risk_service.py` — Risk Scoring Engine

```python
RISK_KEYWORDS = {
    'unlimited liability':       10,
    'indemnify and hold harmless': 8,
    'sole discretion':           7,
    'without notice':            8,
    'irrevocable':               7,
    'perpetual':                 6,
    'waive all rights':          9,
    'non-refundable':            5,
    'automatic renewal':         6,
    'liquidated damages':        7,
    'penalty':                   6,
    'unilateral':                7,
    'governing law':             3,
    'arbitration':               4,
    'force majeure':             3,
}

class RiskService:
    def calculate_risk(self, text):
        text_lower = text.lower()
        total_weight = sum(RISK_KEYWORDS.values())
        score = 0
        matched = []

        for keyword, weight in RISK_KEYWORDS.items():
            if keyword in text_lower:
                score += weight
                matched.append({'keyword': keyword, 'weight': weight})

        normalized = min(round((score / total_weight) * 100, 2), 100)

        if normalized >= 75:   level = 'CRITICAL'
        elif normalized >= 50: level = 'HIGH'
        elif normalized >= 25: level = 'MEDIUM'
        else:                  level = 'LOW'

        return {
            'risk_score':    normalized,
            'risk_level':    level,
            'matched_risks': matched,
        }
```

function ChatPanel({ appointment, onClose, user }) {
  const [messages, setMessages] = useState([]);
  const [input, setInput]       = useState('');
  const [sending, setSending]   = useState(false);
  const bottomRef  = useRef(null);
  const loadingRef = useRef(false); // guard: skip if request already in flight

  const load = async () => {
    if (loadingRef.current) return;
    loadingRef.current = true;
    try {
      const { data } = await appointmentsApi.getMessages(appointment.id);
      setMessages(data.messages || []);
    } catch { } finally {
      loadingRef.current = false;
    }
  };

  useEffect(() => { load(); }, [appointment.id]);
  useEffect(() => { bottomRef.current?.scrollIntoView({ behavior: 'smooth' }); }, [messages]);
  // Poll every 8 seconds — Railway Postgres via external proxy needs breathing room
  useEffect(() => {
    const t = setInterval(load, 8000);
    return () => clearInterval(t);
  }, [appointment.id]);

  const send = async () => {
    if (!input.trim()) return;
    setSending(true);
    try {
      await appointmentsApi.sendMessage(appointment.id, input.trim());
      setInput('');
      await load();
    } finally { setSending(false); }
  };

  return (
    <div className="chat-panel-overlay">
      <div className="chat-panel">
        <div className="chat-header">
          <p>{appointment.lawyer_name}</p>
          <button onClick={onClose}><X size={16} /></button>
        </div>
        <div className="chat-messages">
          {messages.map(m => (
            <div key={m.id} className={`message ${m.sender_id == user?.id ? 'mine' : 'theirs'}`}>
              <span>{m.message}</span>
              <time>{new Date(m.created_at).toLocaleTimeString()}</time>
            </div>
          ))}
          <div ref={bottomRef} />
        </div>
        <div className="chat-input-row">
          <input value={input} onChange={e => setInput(e.target.value)}
            onKeyDown={e => e.key === 'Enter' && send()}
            placeholder="Type a message..." />
          <button onClick={send} disabled={sending}>
            {sending ? '...' : <Send size={16} />}
          </button>
        </div>
      </div>
    </div>
  );
}
